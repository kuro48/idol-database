# コントリビュータープログラム（Contributor Program）

投稿審査を通過した貢献者への報酬制度の仕様です。

---

## 1. 概要

本データベースへの質の高い情報提供を継続してくれた投稿者（コントリビューター）に対して、
将来リリース予定のファン交流 SNS アプリでバッジ・特典を付与します。

**識別子**: 投稿時に登録したメールアドレスを識別キーとします。

---

## 2. バッジ制度

承認された投稿数に応じてバッジが付与されます。

| バッジ | 条件（累計承認件数） | SNS アプリでの特典（予定） |
|--------|-----------------|--------------------------|
| Bronze | 5件以上 | プロフィールにバッジ表示 |
| Silver | 20件以上 | 専用フレーム・限定スタンプ |
| Gold | 50件以上 | 限定機能の先行アクセス |
| Platinum | 100件以上 | 公式コントリビューター認定・特別バッジ |

> バッジは最高ランクのものが表示されます（累積制）。

---

## 3. ポイント計算ルール

### カウント対象

- 承認された投稿のみカウント（pending・rejected・needs_revision はカウントしない）
- 同一アイドルへの重複投稿は1件としてカウント
- 差し戻し後に修正して承認された場合は1件としてカウント

### カウント対象外

- 却下された投稿
- 運営者自身による直接登録

---

## 4. 不正防止ルール

匿名投稿（メールアドレス識別）の性質上、以下の対策を設けます：

| リスク | 対策 |
|--------|------|
| 複数メールアドレスによる水増し | SNS アプリ連携時に同一アカウントへ統合。未連携分は SNS 登録後に申告で統合可能 |
| 低品質な大量投稿 | 承認率が低いメールアドレスは審査優先度を下げる |
| スパム投稿 | 短期間に大量投稿されたメールアドレスは一時停止 |

---

## 5. SNS アプリとの連携設計

将来の SNS アプリとの連携を見越し、API に以下のフィールドを保持します：

```
Contributor エンティティ（Phase 2 で実装予定）:
  contributor_email  : string    // 一意の識別子（例: user@example.com）
  sns_user_id        : *string   // SNS アプリのユーザーID（nullable、連携後に設定）
  approved_count     : int       // 累計承認件数
  total_submissions  : int       // 累計投稿件数
  badge_level        : string    // "none" | "bronze" | "silver" | "gold" | "platinum"
  last_submitted_at  : time.Time // RFC3339形式
  created_at         : time.Time // RFC3339形式
```

SNS アプリ連携の流れ：
```
1. ユーザーが SNS アプリに登録（メールアドレスを使用）
2. SNS アプリが API に連携リクエストを送信
3. contributor_email が一致したら sns_user_id を紐付け
4. 以後はバッジ情報を SNS アプリ側で表示可能に
```

---

## 6. Webhook 通知

SNS アプリとの疎結合な連携のため、以下の Webhook イベントを送信します（既存の Webhook 機能を活用）：

| イベント | タイミング | ペイロード |
|---------|-----------|-----------|
| `submission.approved` | 投稿が承認された時 | contributor_email, badge_level, approved_count |
| `contributor.badge_upgraded` | バッジランクが上がった時 | contributor_email, old_badge, new_badge |

---

## 7. 収益化フェーズとの関係

| フェーズ | 状態 | コントリビューター施策 |
|---------|------|---------------------|
| Phase 1（現在） | API 公開・投稿審査開始 | バッジ制度の設計・データ蓄積開始 |
| Phase 2 | SNS アプリ公開 | バッジ表示・限定機能の付与 |
| Phase 3 | 収益化開始 | プレミアム会員制度・バッジ有償アップグレード（検討） |
| Phase 4 | 事務所パートナー | 公式認定コントリビューターへの特別報酬 |
