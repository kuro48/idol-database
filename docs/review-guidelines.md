# 審査ガイドライン（Review Guidelines）

投稿申請（Submission）を審査する運営者向けのガイドラインです。

## 1. 審査の基本方針

**承認の唯一の基準**: 投稿内容が事務所公式サイトの情報と一致していること。

審査者は以下を行います：
1. 投稿者が提示したソースURL（事務所公式サイト）を実際に開く
2. 投稿内容と公式サイトの内容を照合する
3. 一致していれば承認、相違・不明点があれば差し戻しまたは却下

---

## 2. 審査フロー

```
[pending 一覧] を確認
  │
  ↓
ソースURLを開く（事務所公式サイトを確認）
  │
  ├─ URLが事務所公式サイトではない → 即「却下」
  │
  ├─ URLにアクセスできない → 差し戻し（URL確認を依頼）
  │
  ├─ 内容が公式サイトと一致 → 「承認」
  │
  ├─ 内容に誤りがある → 「差し戻し」（修正箇所を明記）
  │
  └─ 禁止情報を含む → 「却下」（理由を明記）
```

---

## 3. 承認基準

以下を全て満たす場合に承認します：

- [ ] ソースURLが事務所公式サイトのドメインである
- [ ] ソースURLにアクセスでき、内容が確認できる
- [ ] 投稿内容と公式サイトの記載が一致している
- [ ] 禁止情報（本名・住所・憶測など）が含まれていない
- [ ] 既存データと重複していない（または更新内容が明確である）

---

## 4. 却下基準

以下のいずれかに該当する場合は即座に却下します：

| 理由コード | 内容 |
|-----------|------|
| `INVALID_SOURCE` | ソースURLが事務所公式サイトではない（ファンサイト・Wikipedia等） |
| `PROHIBITED_INFO` | 本名・住所・プライベート情報・誹謗中傷を含む |
| `UNVERIFIABLE` | 情報が公式サイトに存在せず確認不能 |
| `RUMOR` | 憶測・噂・未公表情報 |
| `COPYRIGHT` | 著作権保護テキストの直接コピー |

---

## 5. 差し戻し基準と理由テンプレート

差し戻しは「修正すれば承認できる」場合に使います。

### ソースURLの問題

```
ご投稿ありがとうございます。
提示いただいたソースURLは事務所公式サイトではないため、
事務所が運営する公式サイトのURLに変更してください。
詳細: docs/submission-policy.md
```

### URLにアクセスできない

```
ご投稿ありがとうございます。
提示いただいたソースURLにアクセスできませんでした。
正しいURLをご確認の上、再投稿をお願いします。
```

### 内容の不一致

```
ご投稿ありがとうございます。
以下の項目について、公式サイトの記載と異なる点がありました：
- {フィールド名}: 公式サイトの記載をご確認ください
修正の上、再投稿をお願いします。
```

### 必須ソースURLの不足

```
ご投稿ありがとうございます。
投稿内容の確認に必要なソースURLが不足しています。
事務所公式サイトの該当ページURLを追加の上、再投稿をお願いします。
```

## 6. 審査時の注意事項

### 生年月日・出身地

- 公式サイトに記載がない場合は入力不可（差し戻し）
- 出身地は都道府県レベルのみ許可（市区町村以下が入力されていたら差し戻し）

### 画像URL

- 画像URLが事務所公式サイトのドメインから配信されていることを確認
- 外部の画像ホスティングサービス（Imgur等）は不可

### 卒業・引退メンバー

- 現在の公式サイトで「現役メンバー」として掲載されているかを確認
- 卒業済みメンバーを「現役」として登録しないよう注意

### 重複チェック

- 審査前に既存データベースで同名のアイドル・グループを検索
- 重複が見つかった場合は差し戻し（既存データの更新申請として再提出を依頼）

---

## 7. 審査の応答時間目標

| 優先度 | 状況 | 目標応答時間 |
|--------|------|-------------|
| 高 | 削除申請・プライバシー関連 | 24時間以内 |
| 中 | 通常の投稿審査 | 72時間以内 |
| 低 | 大量投稿・バルク審査 | 1週間以内 |

---

## 8. 審査結果の記録

全審査結果（承認・却下・差し戻し）を記録します。記録する情報：

- 審査者ID
- 審査日時
- 判定結果（approved / rejected / needs_revision）
- 却下・差し戻しの理由コード
- 参照したソースURL
