# 法的ガイドライン

## 📋 概要

本プロジェクトは**ユーザー投稿型プラットフォーム**として、プロバイダ責任制限法に基づく法的保護を受けながら運営します。

---

## ⚖️ 法的枠組み

### プロバイダ責任制限法（2001年）

```yaml
適用要件:
  1. 情報の流通を媒介する（プラットフォーム提供者）
  2. 自ら情報を発信していない（ユーザー投稿のみ）
  3. 違法情報を知ったら迅速に削除（24時間以内）

効果:
  - 運営者の損害賠償責任が限定される
  - ただし、完全免責ではない
```

---

## 🚨 法的リスクと対策

### 1. 肖像権侵害

**リスク**:
- ユーザーがアイドルの写真を無断投稿
- 本人や事務所からのクレーム

**対策**:
```yaml
禁止事項:
  - 画像の直接アップロード禁止
  - 外部URL（公式サイト、SNS）のリンクのみ許可

削除対応:
  - 申請から24時間以内に対応
  - 本人確認が必要な場合は48時間以内

投稿時の警告:
  "画像は公式サイトまたは公式SNSのURLのみ設定できます。
   無断転載画像のリンクは禁止されています。"
```

### 2. パブリシティ権侵害

**リスク**:
- アイドルの名前・肖像を商業利用
- 経済的利益の侵害

**対策**:
```yaml
Phase 1-2（無料期間）:
  - 非営利的利用を明記
  - 「百科事典的な情報提供」を強調

Phase 3以降（収益化時）:
  - 事務所との正式契約
  - ライセンス料の支払い
  - 利用規約に商用利用を明記
```

### 3. 著作権侵害

**リスク**:
- プロフィールテキストのコピー
- 公式サイトからの無断転載

**対策**:
```yaml
投稿ガイドライン:
  - 自分の言葉で記述
  - 公式サイトからの直接コピー禁止
  - 引用する場合は出典明記

ライセンス:
  - ユーザー投稿: CC BY-SA 4.0
  - 公式データ（Phase 4以降）: 個別契約
```

### 4. 個人情報保護法

**リスク**:
- 本名、住所などの私的情報の公開
- プライバシー侵害

**対策**:
```yaml
許可する情報:
  ✅ 芸名（活動名）
  ✅ 所属グループ
  ✅ デビュー年
  ✅ 公式リンク

慎重に扱う情報:
  ⚠️ 生年月日（年のみ推奨）
  ⚠️ 出身地（都道府県レベル）

禁止する情報:
  ❌ 本名（公表していない場合）
  ❌ 住所
  ❌ 私的な連絡先
  ❌ プライベート情報
```

## 📝 利用規約の構成

### 必須セクション

```markdown
## 1. サービスの性質
本サービスはユーザー投稿型のアイドル情報データベースです。
運営者はプラットフォームを提供するのみであり、投稿内容について
直接的な責任を負いません。

## 2. 投稿者の責任
投稿者は以下を保証します:
- 著作権・肖像権を侵害しない
- 虚偽情報・誹謗中傷でない
- プライバシーを侵害しない
- 公開されている事実情報である

## 3. 禁止事項
- 著作権侵害画像のリンク
- 本名などの私的情報
- 未確認の噂・憶測
- 誹謗中傷・ヘイトスピーチ

## 4. 運営者の権利
- 違法・不適切なコンテンツの削除
- 投稿内容の編集・修正
- アカウントの停止・削除
- サービス内容の変更

## 5. 削除申請
権利侵害を受けた方は以下から削除申請が可能です:
POST /api/v1/removal-requests

申請から24時間以内に対応します。

## 6. 免責事項
- 情報の正確性は保証しません
- ユーザー投稿による損害の責任を負いません
- サービスの継続性を保証しません

## 7. ライセンス
投稿されたコンテンツは CC BY-SA 4.0 ライセンスで公開されます。
投稿することで、このライセンスに同意したものとみなします。
```

## 🛡️ 削除申請フロー

### 1. 申請受付

```go
type RemovalRequest struct {
    IdolID      string    `json:"idol_id" binding:"required"`
    Requester   string    `json:"requester" binding:"required"`
    // "idol_themself", "agency", "third_party"
    Reason      string    `json:"reason" binding:"required"`
    ContactInfo string    `json:"contact_info" binding:"required,email"`
    Evidence    string    `json:"evidence,omitempty"` // 本人確認資料URL
    Description string    `json:"description" binding:"required"`
    CreatedAt   time.Time `json:"created_at"`
    Status      string    `json:"status"` // pending, approved, rejected
}
```

### 2. 対応プロセス

```yaml
Step 1: 自動受付（即時）
  - リクエストをDBに保存
  - 受付完了メール送信
  - 管理者に通知

Step 2: 内容確認（12時間以内）
  - 削除理由の妥当性チェック
  - 本人確認が必要か判断

Step 3: 削除実行（24時間以内）
  - 該当データの削除
  - 申請者に完了通知
  - ログの記録

Step 4: 事後対応
  - 同一アイドルの再投稿監視
  - 必要に応じてブラックリスト登録
```

### 3. 本人確認方法

```yaml
本人（アイドル本人）:
  - 公式SNSアカウントからのDM
  - 身分証明書の提示
  - 事務所からの委任状

事務所:
  - 公式メールアドレスからの連絡
  - 法人登記情報の確認
  - 代表者の身分証明

第三者:
  - 権利侵害の具体的証明
  - 弁護士からの通知書
```

## 📊 モデレーションポリシー

### 投稿時の自動チェック

```go
type ModerationCheck struct {
    // 必須フィールドチェック
    RequiredFields bool

    // NGワードチェック
    ProhibitedWords bool

    // URLバリデーション
    ValidImageURL bool

    // 重複チェック
    DuplicateCheck bool
}
```

### 手動モデレーション

```yaml
フラグ基準:
  - 通報が3件以上
  - センシティブなワードを含む
  - 新規ユーザーからの投稿
  - 短期間に大量投稿

対応:
  - 24時間以内に内容確認
  - 問題なければ承認
  - 問題あれば削除 or 修正依頼
```

## 🎯 Phase別の法的戦略

### Phase 1-2: リスク最小化

```yaml
方針:
  - 完全にユーザー投稿型
  - 運営者は一切データ収集しない
  - 画像の直接ホスティング禁止
  - 迅速な削除対応

免責事項の強調:
  "本サービスはユーザー投稿型プラットフォームであり、
   運営者は投稿内容の正確性を保証しません。"
```

### Phase 3: 交渉準備

```yaml
実績データ収集:
  - 削除申請への対応実績
  - ユーザー満足度調査
  - 事務所別のポジティブ評価

準備:
  - 弁護士との契約書作成
  - ライセンス契約のひな形
  - パートナーシップ提案書
```

### Phase 4: 公式化

```yaml
契約内容:
  - データ提供範囲
  - 利用期間・更新条件
  - ライセンス料
  - 収益分配モデル
  - 削除・修正権
  - 契約解除条件

移行:
  - ユーザー投稿 → 公式データへ段階移行
  - 過去の投稿者にクレジット表示
  - 編集履歴の保存
```

## 📞 緊急時対応

### 法的クレーム受信時

```yaml
Step 1: 即時確認（1時間以内）
  - 内容の把握
  - 緊急度の判断

Step 2: 弁護士相談（12時間以内）
  - 法的妥当性の確認
  - 対応方針の決定

Step 3: 対応実行（24時間以内）
  - 該当コンテンツの削除
  - 申立人への回答
  - 再発防止策の実施

Step 4: 記録・分析
  - ケースの記録
  - 同様事例の予防策検討
```
